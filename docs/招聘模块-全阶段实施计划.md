## Phase 0 - 紧急主流程修复 ✅

> **目标**：让招聘需求流程符合主流 HR 系统标准——业务部门发起 → 自动触发多节点审批 → HR 基于已审批需求发布职位。

### 0.1 角色系统 ✅
- [x] 新增角色 `R_HR`（HR 管理员）
- [x] 新增角色 `R_DEPT_MANAGER`（部门负责人）
- [x] 新增 Mock 账号：`hr / 123456`（张HR）

### 0.2 RecruitmentDemand 数据模型扩展 ✅

### 0.3 自动审批流 ✅

## ⚠️ 实施注意事项

1. **严格按 CLAUDE.md 开发规范**：
   - 页面直接用中文（不用 i18n 函数）
   - 表格按钮用 `link` 类型
   - 滚动用 `el-scrollbar`
   - CRUD 优先用 `createCrudApi` + `createCrudMock`

2. **Mock 模式必须保持启用**：`VITE_USE_MOCK=true` 不要关闭

3. **不执行 `pnpm dev/build`**：由用户自行操作

4. **子路由路径规则**：
   - 显示在菜单中的：path 不含 `/`（如 `talent-pool`）
   - 隐藏的详情/编辑路由：path 含 `/` + `isHide: true` + `activePath`

5. **角色权限模式**：继续用"角色组合"（admin 叠加多角色），不回到"R_SUPER 自动继承"的上帝视角模式

6. **配置模板引用关系**：
   - Phase 2 的模块升级必须**引用 Phase 1 的模板**，不要重复造数据
   - 例如面试管理的评价表单必须引用 `interviewEvaluationTemplateApi` 的数据
