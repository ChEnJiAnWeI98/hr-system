# 招聘与配置模块 - 竞品分析与功能完整清单

**文档版本**：V1.0
**编制日期**：2026-04-20
**适用对象**：企业HR系统（中大型企业 500~5000人）
**调研目的**：对标主流HR系统"招聘与配置"模块，输出完整功能清单，供产品规划选型

---

## 一、调研说明

### 1.1 调研方法
- 竞品选型：覆盖国内主流招聘SaaS、一体化HR SaaS、以及国际标杆产品
- 分析维度：产品定位、目标客户、核心招聘功能、差异化能力、定价模式
- 信息来源：官方产品文档、公开产品演示、HR业内评测报告

### 1.2 现有系统范围
当前HR系统已实现的招聘模块包含 **6 个子模块**：
1. 招聘需求（demand）
2. 职位发布（job）
3. 简历管理（resume）
4. 面试管理（interview）
5. Offer管理（offer）
6. 入职衔接（onboarding）

需求说明书第十一章定义的功能覆盖：需求管理 / 职位发布 / 简历管理 / 面试安排 / Offer发放 / 入职对接，**但对现代招聘系统的核心能力（AI筛选、人才库、招聘协同、候选人门户、数据分析等）提及较少**。

### 2.2 核心竞品详解

#### 北森招聘云（Beisen Recruit）
- **定位**：国内招聘SaaS TOP 1，覆盖校招+社招+高管招聘全场景
- **核心功能 Top 5**：
  1. 招聘全流程管理（需求→发布→筛选→面试→Offer→入职）
  2. AI简历解析 + 智能匹配（JD标签化匹配候选人）
  3. 人才库（统一人才中心，支持智能推荐、黑名单）
  4. 多渠道职位发布（30+ 招聘网站一键同步）
  5. 候选人体验门户（C端小程序、进度查询、AI对话）
- **优势**：人才测评深度整合（北森测评基因）、校招场景成熟（宣讲会/AI群面）、数据分析强
- **劣势**：价格高、配置复杂、定制化门槛高

#### Moka Recruit
- **定位**：新一代智能招聘管理系统，强调"好用+智能"
- **核心功能 Top 5**：
  1. 全流程招聘协作（HR / 业务 / 面试官 / Boss 多角色）
  2. AI 简历筛选（AI 评分、JD 匹配度打分）
  3. 视频面试 + AI 面试（支持录屏、AI 智能提问、转写分析）
  4. 猎头协作门户（猎头推荐候选人、在线协同）
  5. 数据驾驶舱（招聘漏斗、渠道分析、面试官效率）
- **优势**：交互体验好、移动端完善、AI面试能力领先、价格相对合理
- **劣势**：校招能力弱于北森、缺少深度测评

#### 用友大易（Great Easy Recruit）
- **定位**：智能化招聘管理平台，强调"智能决策"
- **核心功能 Top 5**：
  1. 招聘全流程 + 审批灵活配置
  2. AI 视频面试 + 人岗匹配
  3. 内推激励体系（积分、排行榜、激励金）
  4. 校招场景（ATS校招+在线笔试+群面）
  5. 人才盘点 + 人才地图
- **优势**：与用友NC/U8生态打通、国企客户多
- **劣势**：UI相对传统、移动端略弱

#### e成科技（e-chen.com）
- **定位**：AI招聘+人才画像（Profile-based Recruiting）
- **核心功能 Top 5**：
  1. 人才画像（Talent Profile）自动生成
  2. AI 聊天机器人（HR Bot 主动触达候选人）
  3. 简历智能解析（超过1000个字段）
  4. 内部人才推荐（Internal Mobility）
  5. 招聘知识图谱（技能图谱、公司图谱）
- **优势**：AI 能力领先、人才画像深度
- **劣势**：客户规模相对小、品牌力弱于北森/Moka

#### SAP SuccessFactors Recruiting
- **定位**：全球化一体化HCM中的招聘模块
- **核心功能 Top 5**：
  1. 全球多语言 / 多法律实体职位发布
  2. 候选人关系管理（CRM-like 招聘CRM）
  3. 招聘营销（招聘官网、Landing Page、SEO）
  4. 高度可配置审批流与权限体系
  5. 与 SAP Core HR / LMS / Performance 深度集成
- **优势**：国际化能力、合规性（GDPR/EEO）、企业级可扩展
- **劣势**：实施周期长、本土化弱、价格昂贵

#### Workday Recruiting
- **定位**：云原生现代HCM的招聘模块
- **核心功能 Top 5**：
  1. 统一数据模型（候选人=员工未来态）
  2. 招聘仪表盘 + 内置 BI
  3. 面试官协同评估
  4. 移动端优先体验
  5. 招聘营销 + CRM
- **优势**：UI现代、数据分析能力强、北美市占率高
- **劣势**：中国本土化有限、中小企业难用

## 六、落地建议与优先级规划

### 6.1 分期建设路径

**第一期（1~2个月，完善ATS基础）**
- 简历智能解析、查重、人才库、淘汰原因字典
- 面试评价模板、多人协同打分
- Offer 模板引擎、候选人笔记
- 招聘漏斗报表基础版

**第二期（2~3个月，协同与中台化）**
- 内推体系、渠道管理（即便是手工登记也先做配置）
- 招聘官网/候选人H5门户
- 电子签集成、Pre-onboarding
- 渠道 ROI / 招聘周期分析

**第三期（3~6个月，AI与增值）**
- AI JD匹配、AI 视频面试
- 背调/测评集成
- 自动化规则引擎
- 校招场景（如有需要）

### 6.2 差异化建议

鉴于你的产品定位是 **中大型企业一体化HR**（而非招聘专精SaaS），建议：

- **深度做好 ATS 基础**（P0 + P1），达到行业平均水平
- **发挥一体化优势**：招聘与员工档案、编制、薪酬、绩效深度打通（北森/Moka 也在做但相对弱）
- **AI 能力选择性接入**：优先做 "简历解析 + 智能匹配"，AI 面试可先不做（ROI 较低）
- **不必追求 30+ 渠道**：先支持主流 3~5 个渠道（Boss直聘、猎聘、智联、内推、官网）即可

### 6.3 建议不做的（ROI 低）

- 校招全流程（若你的目标客户是以社招为主的中大型企业）
- 招聘营销 / SEO 等 C 端功能
- 海外/多语言/多法律实体（除非明确出海客户）
- 复杂测评体系（可走集成而非自研）
