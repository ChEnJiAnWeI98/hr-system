## 一、总体排期与里程碑

| Phase | 主题 | 周期 | 开始条件 | 里程碑 |
|---|---|:---:|---|---|
| **Phase 1** | 打地基：数据字典 / 员工档案 / 组织 / 岗位 / RBAC | 2~3 周 | V2.0 方案确认 | ✨ 地基层可独立运行，员工档案 8 Tab 可查看 |
| **Phase 2** | 菜单归位 + 已有模块迁移 | 1 周 | Phase 1 完成 | ✨ 21 一级 → 8 一级，旧模块接入新员工池 |
| **Phase 3** | 核心人事补齐 + 考勤假期 | 2 周 | Phase 2 完成 | ✨ 异动/离职/编制 + 完整考勤 |
| **Phase 4** | 薪酬福利 + 培训与发展 | 2~3 周 | Phase 3 完成 | ✨ 完整薪酬闭环 + 培训管理 |
| **Phase 5** | 数据洞察 + AI 中枢 + 系统完善 | 1~2 周 | Phase 4 完成 | ✨ 跨模块洞察 + 完整 RBAC |
| **Phase 6** ⭐ | **终态结构优化：菜单拆分 + 命名统一** | 0.5 周 | Phase 5 完成 + Diff 审查完成 | ✨ 8 → 10 一级菜单（对标北森/金蝶），菜单最终稳定 |

**总周期：10~11 周**（含 Review 时间）

**Review 节点**：每 Phase 结束停下，用户 Review 后方可进入下一 Phase。

### ⚠️ Phase 6 的必要前置条件

Phase 6 菜单拆分**必须在 Phase 5 和 Diff 审查完成后才能进行**，理由：
1. **真实页面数未定**：重构中期各一级下存在新旧重叠页面，Diff 审查后会合并/废弃一批 → 最终页面数才是拆菜单的依据
2. **业务边界未清**：Phase 4 的"人才发展"（人才盘点/能力模型/继任）Phase 5 的"数据洞察"等模块业务边界要完全稳定后才能拆
3. **工作量避免翻倍**：结构优化应"一次做对"，避免 Phase 期间多次调整路由/i18n/RBAC

### Phase 6 目标菜单（对标北森/金蝶 10 一级）

```
01 组织人事      07 绩效管理（拆自原 05）
02 招聘与入职    08 人才发展（拆自原 05：盘点+能力+继任）
03 考勤与假期    09 培训管理
04 薪酬管理（拆自原 04）   10 数据洞察
05 社保公积金（拆自原 04）   11 系统管理
06（可复用）     → 总计 10 一级（+工作台）
```

## 九、执行原则（给 Claude 的 SOP）

这一章是给 Claude 自己看的执行原则，确保 10 周过程不跑偏。

### 9.1 每个 Phase 的 SOP

1. **开始前**：读完架构篇 + 迁移篇相关章节
2. **动手前**：创建 Phase 专属的 plan（类似绩效 Phase A~D 的模式），用户 Review 通过再动
3. **实施中**：用 TaskCreate / TaskUpdate 跟踪所有子任务
4. **遇到决策**：停下问用户，不擅自决定
5. **每个子任务完成**：更新 plan 状态 + 简短口头汇报
6. **Phase 结束**：等待用户 Review 确认，再进入下一 Phase

### 9.2 代码层 SOP

1. **先地基后业务**：Phase 1 的地基做不完，绝不动 Phase 2 的业务
2. **统一组件优先**：遇到组织/员工/岗位选择，必须用统一组件
3. **员工数据只读 store**：所有业务页面员工数据必须 `useEmployeeStore()`
4. **状态机优先**：有状态变化的业务，先设计状态机再写 UI
5. **字典读取统一入口**：`useDictStore().get(dictCode)`
6. **写代码前必读 CLAUDE.md**：每一次写新 Vue 页面都要核对布局规范 / 按钮规范 / 表格规范

### 9.3 Mock 数据 SOP

1. **禁止用 Math.random**：所有 Mock 数据基于种子确定性生成
2. **统一员工池**：`src/mock/core/employeePool.ts` 是唯一员工数据源
3. **数据有代表性**：预置数据要覆盖各种状态 / 角色 / 场景
4. **日期用固定值**：所有 Mock 日期用 2026 年的固定日期，不用 `new Date()`

### 9.4 Plan 文档 SOP

每个 Phase 结束，在配套的 plan 文档里：
- 标记 ✅ 完成
- 增加 "变更记录" 版本（V1.x → V1.(x+1)）
- 记录本 Phase 的关键决策 / 坑 / 偏差
